### 题目描述
某涉密单位下发了某种票据，并要在年终全部收回。

每张票据有唯一的ID号。

全年所有票据的ID号是连续的，但ID的开始数码是随机选定的。

因为工作人员疏忽，在录入ID号的时候发生了一处错误，造成了某个ID断号，另外一个ID重号。

你的任务是通过编程，找出断号的ID和重号的ID。

假设断号不可能发生在最大和最小号。

#### 输入格式

第一行包含整数 $N$，表示后面共有 $N$ 行数据。

接下来 $N$ 行，每行包含空格分开的若干个（不大于100个）正整数（不大于100000），每个整数代表一个ID号。

#### 输出格式

要求程序输出1行，含两个整数 $m,n$，用空格分隔。

其中，$m$表示断号ID，$n$表示重号ID。

#### 数据范围

$1 \le N \le 100$

#### 输入样例：

```
2
5 6 8 11 9
10 12 9
```

#### 输出样例：

```
7 9
```

---
### 算法
排序 模拟


#### C++ 代码
```
#include <bits/stdc++.h>
using namespace std;
const int N = 10010;
int a[N];
int res1,res2;
int cnt;
int main () {
    int n;
    cin>>n;
    string line;
        getline(cin,line);
    while(n--){
        getline(cin,line);
        stringstream ssin(line);
        while(ssin>>a[cnt])cnt++;
    }
    
        sort(a,a+cnt);
        for(int i=1;i<=cnt;i++){
            if(a[i]==a[i-1])res1=a[i];
            else  if(a[i]>=(a[i-1]+2)) res2=a[i]-1;
        }
        cout<<res2<<" "<<res1<<endl;
    return 0;
}
```
#### 反思
因为这道题需要读入很多行的数据，并且这些行中每一行的数据的数目是不知道的，所以使用了getline来读入每一个数据，然后在后面使用stringstream来流输出，需要注意一下写法，以后说不定可以用到，需要注意两个问题，第一个就是使用getline读入的时候会读入所有的东西，因为在前面使用cin>>来读入n，在最后有一个回车（\n）后面使用的getline也会读入这个回车的符号，所以在正式读入每一行的数据之前需要先使用一个getline来读入这个回车符号。第二就是需要注意一下ssin这个东西的作用域，ssin的作用域是在循环内部，离开了这个循环ssin就无法使用了，这两点需要稍微注意一下。
