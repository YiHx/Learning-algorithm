### 题目描述
小明这些天一直在思考这样一个奇怪而有趣的问题：

在 $1 \sim N$ 的某个排列中有多少个连号区间呢？

这里所说的连号区间的定义是：

如果区间 $[L, R]$ 里的所有元素（即此排列的第 $L$ 个到第 $R$ 个元素）递增排序后能得到一个长度为 $R-L+1$ 的“连续”数列，则称这个区间连号区间。

当 $N$ 很小的时候，小明可以很快地算出答案，但是当 $N$ 变大的时候，问题就不是那么简单了，现在小明需要你的帮助。

#### 输入格式

第一行是一个正整数 $N$，表示排列的规模。

第二行是 $N$ 个不同的数字 $P_i$，表示这 $N$ 个数字的某一排列。

#### 输出格式

输出一个整数，表示不同连号区间的数目。

#### 数据范围

$1 \le N \le 10000$,  
$1 \le P_i \le N$

#### 输入样例1：

```
4
3 2 4 1
```

#### 输出样例1：

```
7
```

#### 输入样例2：

```
5
3 4 2 5 1
```

#### 输出样例2：

```
9
```

#### 样例解释

第一个用例中，有 $7$ 个连号区间分别是：$[1,1], [1,2], [1,3], [1,4], [2,2], [3,3], [4,4]$  
第二个用例中，有 $9$ 个连号区间分别是：$[1,1], [1,2], [1,3], [1,4], [1,5], [2,2], [3,3], [4,4], [5,5]$

---
### 算法
枚举

#### C++ 代码
```
#include <bits/stdc++.h>
using namespace std;
const int N = 10010;
int a[N];
int res;
int main () {
   int n;
   int maxn,minn;
   cin>>n;
   for(int i=0;i<n;i++){
       cin>>a[i];
   }
   for(int i=0;i<n;i++){
        maxn=0;
        minn=10010;
       for(int j=i;j<n;j++){
           
           maxn=max(maxn,a[j]);
           minn=min(minn,a[j]);
           if((maxn-minn)==(j-i))res++;
           
       }
   }
   cout<<res<<endl;
    return 0;
}
```
#### 反思
这道题最重要的是能够巧妙的发现如果是一个连号区间，那么这个区间的最大值减去最小值的大小会等于这个区间的起始的数的下标和结束的位置的数字的下标的差，还有就是这道题可以很巧妙的在枚举每一个新增的数的时候来计算一下当前的这个区间的最大值和最小值
